# Build or Bust

基于 Unreal Engine 5.8 的双人同场竞技据点防守第一人称射击游戏。

## 项目简介

两名玩家进入同一战场，各自击杀来袭的丧尸获取分数，同时共同守护场地中央的核心。当全部 10 波进攻结束、或核心被摧毁时，依据双方得分判定胜负。项目在 UE5.8 第一人称模板基础上扩展，核心玩法与网络同步使用 C++ 实现。

## 主要特性

- **多人同场竞技**：基于监听服务器（Listen Server）的实时联机，两名玩家各自计分并在结算时比较。
- **波次系统**：共 10 波，敌人数量与强度随波次递增。
- **三种敌人**：普通型、快攻型（高速低血）、肉盾型（高血低速），击杀得分与敌人强度挂钩。
- **增益系统**：升级时从三个候选中选择一项战斗增益（强化弹药、处决、霜冻弹、剧毒弹、迅捷步伐等）。
- **据点防守**：中央核心可被敌人攻击，血量上限随波次提升；核心被摧毁则对局结束。
- **关卡地形**：三向走廊布局，配合掩体与弹跳台，提供立体走位空间。
- **Android 版本**：提供打包好的 APK，支持虚拟摇杆与触屏视角操作。

## 运行环境

- Unreal Engine 5.8
- Windows 10 / 11
- Visual Studio 2022（编译 C++ 模块所需）

## 构建与运行

1. 克隆后拉取 LFS 资源：
   ```bash
   git lfs install
   git lfs pull
   ```
2. 右键 `BuildOrBust.uproject` → Generate Visual Studio project files，使用 VS 编译；或直接用 UE5.8 打开工程，首次打开按提示编译。
3. 在编辑器中打开主关卡 `Content/Variant_Shooter/Lvl_Shooter`，点击 Play 运行。
4. Android：Releases 页面提供 `BuildOrBust-arm64.apk`（Android 8.0 及以上，arm64 设备），下载安装即可。

### 多人测试

在 Play 下拉设置中配置：

- Net Mode：`Play As Listen Server`
- Number of Players：`2`

## 玩法规则

- 击杀丧尸获得分数，敌人越强得分越高。
- 守住中央核心，核心血量归零则全场失败。
- 撑过全部 10 波后由得分较高者获胜；核心被摧毁时同样依据当前得分判定胜者。

## 项目结构

```
Source/BuildOrBust/          游戏 C++ 源码
├─ WaveManager               波次生成与计分
├─ BaseCore                  中央核心（血量 / 受击 / 波次成长）
├─ UpgradeComponent          增益抽取与应用
├─ BoBJumpPad                弹跳台
└─ Variant_Shooter/          角色、武器、AI、HUD、GameMode / GameState
Content/Variant_Shooter/     关卡、蓝图、UI、字体等资源
Config/                      项目与输入配置
```

## 技术实现要点

- **网络同步**：得分使用复制的 `PlayerState` 分别记录；开火通过 Server RPC 在服务器端结算伤害，敌人死亡状态复制到客户端，保证命中与击杀对所有客户端一致。
- **波次与 AI**：`WaveManager` 控制刷怪节奏与数值缩放，敌人 AI 寻路至核心并发起攻击。
- **HUD**：通过 `AHUD::DrawHUD` 为每名玩家绘制目标、波次、核心血量与实时比分；中文使用离线烘焙的距离场字体以保证清晰度。

## 已知问题与后续规划

- 敌人沿用模板的持枪动画，缺少徒手 / 近战动作资源。
- 增益仅实现数值效果，命中及状态（霜冻、中毒）的粒子特效尚未制作。
- 美术以灰模与占位材质为主，暂无正式美术与音效。
- 目前为单机进程内双人（PIE），未实现跨设备局域网联机；移动端为单机体验。
- 结算界面有待进一步完善。
